Definition: A monoid is a type $T$ with binary operation $\oplus$ which combines its values.

  • $\oplus$ must be associative: $(a \oplus b) \oplus c = a \oplus (b \oplus c)$.
  • $T$ has a unity element $e$ such that $e\oplus x = x\oplus e = x$ for all $x\in T$.

Definition: A functor is a class template (Functor<T>) with a single template parameter and a callable (map) which has the following properties:

  • map(f, a) is a legal expression when:
    • a is a value of type Functor<T> for some type T.
    • f is a callable that accepts a single argument of type T.
    • Let U be the return type of f(t) for t$\in$T. The result of map(f, a) is type Functor<U>
  • map(g, map(f, a)) = map(gf, a), where 
    
      auto gf = [f, g](auto t) { return g(f(t)); }
